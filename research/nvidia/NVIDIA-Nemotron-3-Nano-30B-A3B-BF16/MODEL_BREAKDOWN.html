Terms you actually need

Parameter

One learned number. Most are BF16, so usually 2 bytes each. Compression must preserve these exactly.

Tensor

A named block of parameters with shape and dtype, like backbone.embeddings.weight.

Shape

The dimensions of a tensor. Shape decides whether row, column, tile, or expert-level transforms make sense.

Dtype

The storage format. BF16 has two bytes per value, and each byte may compress differently.

Hidden size

The internal token-vector width. Here it is 2688, so many matrices connect to 2,688-wide vectors.

Expert

A feed-forward block inside MoE. Experts dominate storage and are the first serious compression surface.

MoE is the main target

MoE means Mixture of Experts. Each MoE layer has 128 routed experts, 1 shared expert, a router gate, and a norm. The router chooses 6 routed experts per token, but every expert still has to be stored exactly.

Tensor patternDtypeShape
mixer.experts.N.up_proj.weightBF16[1856, 2688]
mixer.experts.N.down_proj.weightBF16[2688, 1856]
mixer.shared_experts.up_proj.weightBF16[3712, 2688]
mixer.shared_experts.down_proj.weightBF16[2688, 3712]
mixer.gate.weightBF16[128, 2688]

Layer 1 alone takes 2,594,936,576 bytes. This is why MoE experts are the first serious compression surface.

Related because

Mamba explains regular dense structure

Mamba2 layers are much smaller than MoE layers, but they are clean dense matrices. They are good early probes for BF16 byte statistics, row deltas, column deltas, and exact chunk compression.

Verified layer 0

in_proj.weight: [10304, 2688]

conv1d.weight: [6144, 1, 4]

out_proj.weight: [2688, 4096]

Total with norm: 77,490,048 bytes.

Keep an eye on this

Attention matters more later

Attention is not the storage bottleneck here. It matters for runtime memory because generation stores a key/value cache that grows with context length.

Expected shapes

q_proj.weight: [4096, 2688]

k_proj.weight: [256, 2688]

v_proj.weight: [256, 2688]

o_proj.weight: [2688, 4096]

Loading path

read config.json
  -> load custom NemotronH config
  -> load custom NemotronH model code
  -> read model.safetensors.index.json
  -> map tensor names to shard files
  -> load tensors into matching parameters

Local config and tokenizer loading work. Full model construction is blocked by missing mamba-ssm, and full weight loading is blocked by missing shards 2 through 13.
